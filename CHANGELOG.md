# Changelog

Version history and decision rationale are collected here.

## 1.1.0 (2026-07-18)

- **`economy-checkpoint.js` — split the auto-pin tier by agent.** `Explore` (pure read-only lookup) is now pinned to `model: haiku`, the cheapest tier. `general-purpose` (and the empty `subagent_type` default, which resolves to general-purpose) keeps the `model: sonnet` floor, since it may involve multi-step work beyond mechanical search. Previously both were pinned to `sonnet`. The `additionalContext` note now tells the model which floor it landed on and that `model: opus` (or `model: sonnet` from the haiku floor) is available if the task needs more reasoning.

## 1.0.0 (2026-07-05)

First release. Brings the generator/arbiter split — first built into the spec-driven-development plugin, where it governed a structured workflow — out into **everyday agent work**, via two hooks and no slash commands. The concept: the **main agent is the arbiter** (reviewer, concept-challenger, decision-maker, coordinator) and **subagents are the executors** that do the real work on a cheaper tier. So the premium session model is spent on judgment, and the doing is delegated down. It deliberately says nothing about how you read files.

- **SessionStart hook (`session-start-economy.js`)** — injects a short, always-on reminder of the role split: stay the arbiter, hand execution (searching, file sweeps, generating code/docs) to a subagent on a cheaper tier, and spend premium tokens judging and challenging what comes back. Carries the absolute path to `references/economy-playbook.md` (via `${CLAUDE_PLUGIN_ROOT}`). Chosen over a skill because a skill under-triggers in casual conversation; SessionStart is the reliable always-on lever.
- **PreToolUse hook (`economy-checkpoint.js`)** — the enforceable slice. On a subagent spawn (matcher `Task|Agent`) targeting a built-in agent that inherits the session model (`Explore` / `general-purpose`, or an empty `subagent_type`) with no `model` pinned, it rewrites `tool_input` via `updatedInput` to add `model: sonnet` and injects an `additionalContext` note to re-issue with `model: opus` if the task needs cross-file reasoning. Non-blocking (never `deny`), respects deliberate model choices, never touches custom agents (own `model:` frontmatter), and sets no `permissionDecision` so it doesn't bypass the user's Task permission settings. Fail-open on any parse/shape error.
- **`references/economy-playbook.md`** — the split in full: the role division (arbiter vs executor), the tier default (spawn cheap, upgrade to opus only for genuine cross-file reasoning), the critic role (a cheap executor is safe only with a premium critic), what the hook enforces, and the failure modes — including *executing it yourself* on the premium tier, the one the hook can't catch. Scoped to who executes and at what tier; no reading-method rules.
- **Design notes** — the hook enforces only the tier of exploration subagents at the spawn; the rest of the split (delegating execution at all, staying the arbiter, challenging the output) is guidance, since there is no clean trigger to hook "the main agent did the work itself." The PreToolUse hook can't read the session model (no such field on its stdin), so it always pins the `sonnet` floor: a no-op when the session is already cheap, a real saving only on a premium tier. The hooks never call any MCP and assume no other plugins are installed.
